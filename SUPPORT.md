# Support

Thank you for using Melodic Software projects! This document explains how to get help.

## Getting Help

### GitHub Discussions

For questions, ideas, and general conversation:

- **[Organization Discussions](https://github.com/orgs/melodic-software/discussions)** - Best place for Q&A and community interaction
- **Q&A Category** - Ask questions and get answers from maintainers and community
- **Ideas Category** - Share and discuss feature ideas before creating issues

### Documentation

- Check the repository's `README.md` for setup and usage instructions
- Review the `docs/` folder if present for detailed documentation
- See `CONTRIBUTING.md` for development guidelines

### Bug Reports

If you've found a bug:

1. **Search existing issues** to avoid duplicates
2. **Open a new issue** using the Bug Report template
3. **Provide details**: steps to reproduce, expected vs actual behavior, environment info

See the specific repository's issue tracker to file a bug report.

## Security Issues

**Do NOT open public issues for security vulnerabilities.**

Please review our [Security Policy](SECURITY.md) for responsible disclosure instructions.

## Contact

For business inquiries, partnership opportunities, or matters that don't fit the above channels:

- **Email**: [info@melodicsoftware.com](mailto:info@melodicsoftware.com)
- **Website**: [melodicsoftware.com](https://melodicsoftware.com)

## Response Times

| Channel | Expected Response |
|---------|-------------------|
| Security issues | Within 48 hours |
| Bug reports | Within 7 days |
| Feature requests | Triaged weekly |
| Discussions | Best effort, community-driven |

## What NOT to Use Support For

- **Feature requests** - Use the Feature Request issue template instead
- **Security vulnerabilities** - Follow the Security Policy
- **Pull request questions** - Comment directly on the PR
