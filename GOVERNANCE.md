# Governance

This document describes how melodic-software projects are governed. It is an org-wide default; a repository may override it with its own `GOVERNANCE.md` when its governance differs.

## Roles

- **Maintainers** — own project direction, review and merge changes, and manage releases. melodic-software is currently maintained by [@kyle-sexton](https://github.com/kyle-sexton).
- **Contributors** — anyone who opens an issue or pull request.

## Decision-making

Decisions are made by the maintainers. Significant or hard-to-reverse decisions are recorded in the affected repository (for example, as Architecture Decision Records) so the rationale is durable.

## Changes to governance

Changes to this document are made by the maintainers through the normal pull-request process.
